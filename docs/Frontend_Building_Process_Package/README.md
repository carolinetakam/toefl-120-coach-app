# Frontend Building Process Package

Date: 2026-06-02

Purpose: give an app-building agent the frontend process, design philosophy, and production standard to use when building one of Caroline's apps.

This is not a general architecture-building template. It is a practical frontend build package inspired by the process and philosophy demonstrated in Myriad.

## Start Here

Give the agent this file first:

`agent-handoff/FRONTEND_AGENT_HANDOFF.md`

Then have it read:

1. `process/FRONTEND_BUILD_PROCESS.md`
2. `reference/MYRIAD_FRONTEND_PHILOSOPHY.md`
3. `checklists/FRONTEND_ACCEPTANCE_CHECKLIST.md`

## What This Package Teaches

The agent should build frontend apps that are:

- real product workspaces, not landing pages
- calm, dense, premium, and operational
- human-agency preserving
- production-ready for users
- truthful about data, state, and actions
- built from the existing app instead of rebuilt blindly
- verified on desktop and mobile

## Myriad-Inspired Standard

Myriad demonstrates the target pattern:

- first screen is the useful app
- left sidebar or clear app shell
- visible business/product context
- compact dashboard/workflow density
- source/data status visible
- tactile controls and soft raised panels
- real loading, empty, error, and permission states
- no fake production actions
- agent/AI work is explainable and governable

## When To Use

Use this package when an agent is actively building frontend for:

- SaaS app
- dashboard
- internal tool
- AI product
- productivity app
- command center
- workflow app
- business operating app

Do not use it for marketing-only landing pages unless the app truly needs a landing page.
